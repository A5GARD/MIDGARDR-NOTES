> For any new dev, if your wondering what home is, its referring to your specific users "home" folder. More of a linux term, but the folder containing items like desktop, documents, downloads, favorites and etc. IF you cannot currently see the folder, it is because it is currently hidden from your view, right click anywhere in the file explrer and click on properties.
>
> You will be presented with a very small page, towards the bottom there will be a section called... attributes I think, and there will be two checkboxes the first being read only, which is always checked off and the second labeled hidden, which is never checked off. Click on the check box in order for you to now start viewing hidden items within this folder.
>
> Just a heads up, a window will pop up, giving you two options. If you have a slower computer do not select the recursive option labeled, "Apply changes to this folder, subfolders and files", your computer will not like it depending on where you triggered it from. If you want this to be a quick process whether you have a good computer or not, select on the first option because it will only apply this setting to the folder you are currently in. Where as the second, will do the current folder PLUS, every single folder that is connected to it down the remainder of the file tree. I would never suggest triggering that in say, your c folder as it will take a while to complete, if it doesn't crash your os in the process.
