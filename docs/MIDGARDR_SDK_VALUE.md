The single biggest risk with a 2,800+ component library isn't quality — it's discoverability and documentation. The reason it's hard is that "low floor, high ceiling" sounds simple but the two goals are genuinely in tension with each other. Most libraries pick a side. Radix picks the ceiling — powerful, unstyled, but a junior dev has no idea where to start. Bootstrap picks the floor — you're productive in 20 minutes but a senior dev hits the walls fast and starts fighting the library. What makes it possible at all is the insight that the floor and ceiling aren't actually about the same dimension. The floor is about clarity of the happy path — the default behavior should be so obvious that someone with minimal context can produce something real. The ceiling is about depth of control — every default should be an overridable decision, not a constraint.
